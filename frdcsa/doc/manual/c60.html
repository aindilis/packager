Overview

Although the primary emphasis of RADAR is clustering, it requires a sophisticated interface to a back end to store the information it requires. Since RADAR is primarily used for locating upstream sources, this means that it differs in design requirements from the software repository which is used to track all application information, from locally developed software, preexisting packages, and upstream sources. So it inherits the metadata object model of the general software repository.
